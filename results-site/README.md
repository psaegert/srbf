# Results site (github.io frontend)

This directory is the **standalone results frontend** deployed to GitHub Pages at
<https://psaegert.github.io/srbf/>. It is deliberately **not** part of the MkDocs documentation
(the docs live at <https://srbf.readthedocs.io/>): the results site is a polished, client-side
explorer for the benchmark numbers, nothing more.

## Files

| File | Purpose |
|------|---------|
| `index.html` | The page shell (header, hero, explorer mount, the full "About these numbers" methodology section, the "Paired comparisons" methodology section (`#paired`), the "Metrics" reference with KaTeX-rendered formulas, privacy note, footer). |
| `styles.css` | Standalone styling (light/dark aware), including the prose/metric-card styles for the About and Metrics sections. |
| `explorer.js` | Reads `window.RESULTS_DATA` and renders the 2×2 view registry (Curves / Paired Δ / Table / Matrix, deep-linked as `?view=curves\|paired\|table\|matrix`): the Plotly line + CI-band chart with axis / metric / benchmark / series controls, per-series colour customisation, and the two-tier metric menu; the paired/Table views lazy-fetch `paired_data.json` on first use, and matrix cells tap/click-pin their full record below the table. No server, no PNGs. |
| `results_data.js` | The data: `window.RESULTS_DATA = {metrics, axes, colors, series_meta, records}`, generated by `srbf.analysis.export_data` and augmented by the pipeline. |
| `paired_data.js`, `paired_data.json` | The paired + Table payload, built by flash-ansr's `experimental/eval/build_paired_data.py`: paired per-expression comparison records, Δ(t) curves, budget-grid verdicts, and the absolute Table block — all statistics precomputed in Python (the site only renders). |
| `srbf-icon.svg`, `apple-touch-icon.png`, `srbf-social.png` | Brand assets (favicon, touch icon, `og:image` social card); canonical sources live in `../assets/brand/`. |

## Data contract

- `metrics[*].tier` (`"main"` / `"more"`) drives the two-tier metric dropdown (main = vNRR +
  expression length ratio).
- `colors` maps each series label to its default hex — the **paper palette** (the blue size ramp +
  distinct baseline hues from the scaling notebooks / poster).
- `series_meta` maps each series to `{group, default_on, is_ablation}`: groups render as labelled
  pill rows; `is_ablation: true` series collapse into a `<details>` block, deselected by default.
- Users can recolour any series via its swatch; overrides persist in a single first-party functional
  cookie (`srbf_colors`, written only on an explicit change) with per-series ↺ and a global
  "Reset all colours" that deletes it. The privacy note on the page documents this (GDPR/ePrivacy:
  functional, no consent banner required).

## Regenerating the data

`results_data.js` (and a sibling `results_data.json`) are produced by
`flash-ansr/experimental/eval/build_results_data.py`, which reads the archived evaluation snapshots,
scores them, and calls `srbf.analysis.export_data(..., <this dir>/results_data.json)`. Re-run it
whenever the underlying results change.

`paired_data.js` / `paired_data.json` are produced by
`flash-ansr/experimental/eval/build_paired_data.py`
(`PYTHONPATH=~/Projects/srbf/src python experimental/eval/build_paired_data.py`, run from the
flash-ansr checkout), which needs the same archived evaluation snapshots, writes both files
directly into this directory, and self-checks the Table view's values against `results_data.js` —
so regenerate `results_data.js` first.

## Deploying

The site is pushed to the `gh-pages` branch (GitHub Pages serves it):

```bash
ghp-import -n -p -f -m "deploy results site" results-site
```

(`-n` writes `.nojekyll`.) There is no CI job for this yet; deploy is manual.
